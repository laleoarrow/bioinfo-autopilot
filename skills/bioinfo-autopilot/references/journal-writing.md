# Journal Writing Filter

Use when turning validated bioinformatics or biomedical analyses into manuscript-ready prose, review/article drafts, title/abstract candidates, or figure legends.

## Core Principle
Let evidence drive style. Do not let elegance outrun support.

For clinical manuscripts, use [clinical-accuracy.md](clinical-accuracy.md) first so the style pass happens only after the numbers and guideline contract are locked.
For full manuscripts, use [manuscript-coherence.md](manuscript-coherence.md) before and after the style pass so the paper stays internally consistent.

## Default Modes
- Original research: state the answer first, then the evidence, then the boundary.
- Narrative review or perspective: ask a focused question and synthesize by concept, not paper-by-paper recitation.
- Systematic review: mirror the target journal's required abstract and evidence hierarchy; report search, study counts, and quality before interpretation.

## Style Filters by Journal Family
### Nature, Cell, and Science
- Use simple prose, short paragraphs, and a strong lead sentence.
- Prefer concept-first framing and broad relevance.
- Keep the prose accessible to readers outside the immediate subfield.
- Strip jargon, self-congratulation, and excess qualifiers.
- Keep one idea per sentence when possible.
- Make the take-home message obvious within the first few lines.

### Science / AAAS
- Write for a broad scientific audience, not only the narrow expert group.
- Favor clear, concise, audience-aware explanation over dense technical display.
- Use the framing that helps a careful but non-specialist reader follow the logic.

### Medical Top Journals
- Keep claims clinically anchored and evidence-weighted.
- Use conservative causality unless the design justifies stronger language.
- Match the article type exactly: structured abstract, required headings, and balanced interpretation.
- For reviews, separate established facts, uncertainty, and practice implications.

## Sentence-Level Defaults
- Prefer `suggests`, `is consistent with`, or `supports` over `proves`, `demonstrates`, or `breakthrough`.
- Use numerals, denominators, confidence intervals, and exact counts whenever available.
- Open each paragraph with the claim it is meant to support.
- Remove filler adjectives and repeated concepts.
- Use active voice when it improves clarity; use passive voice only when it is cleaner or more objective.

## Review Article Template
1. Focus the question.
2. Define the evidence base.
3. Synthesize by mechanism, phenotype, or clinical decision point.
4. End each section with what is established, uncertain, and actionable.
5. Keep the conclusion proportional to the strongest evidence.

## Quick Editing Checks
- Does the first sentence tell the reader why this paragraph exists?
- Would a non-specialist in the field still understand the core claim?
- Does every number have context?
- Does the conclusion overreach the evidence?
- Would the paragraph still read cleanly if the strongest adjective were removed?
- Would the same claim still sound equally strong in the abstract, introduction, results, discussion, and conclusion?

## Micro Examples
- Overclaim -> calibrated claim:
  - `These data prove the mechanism.` -> `These data support a mechanistic model.`
- Jargon-heavy -> accessible:
  - `We interrogated the transcriptional landscape.` -> `We analyzed gene-expression changes.`
- Topic dump -> focused review question:
  - `This review discusses obesity.` -> `This review asks which interventions produce durable weight loss.`
